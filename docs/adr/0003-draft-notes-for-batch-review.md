# ADR 0003: GitLab Draft Notes API для Batch Review

**Status:** Accepted

## Context

Нужно реализовать Batch Review — накопление inline-комментариев с последующей единовременной публикацией (как в GitLab Web UI). Альтернатива: хранить черновики локально и публиковать пачкой через обычный Discussions API.

## Decision

Используем **GitLab Draft Notes API** (`/merge_requests/:iid/draft_notes`) для хранения черновиков на стороне сервера.

## Rationale

- Черновики видны пользователю в GitLab Web UI — он может начать ревью в TUI, продолжить в браузере
- Черновики сохраняются между сессиями — не теряются при закрытии утилиты
- Публикация одним вызовом: `POST /draft_notes/bulk_publish`
- Поведение идентично тому, что ожидают пользователи от GitLab-ревью

## Trade-offs

Локальное хранение черновиков было бы проще в реализации, но создавало бы расхождение состояния между TUI и Web UI. Draft Notes API — правильная абстракция для этой задачи, несмотря на то что она менее известна.
