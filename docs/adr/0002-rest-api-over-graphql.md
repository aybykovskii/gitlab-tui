# ADR 0002: REST API через @gitbeaker/node вместо GraphQL

**Status:** Accepted

## Context

GitLab поддерживает оба API. GraphQL позволяет получить MR + diff + комментарии + треды за один запрос. REST требует нескольких запросов, но имеет зрелую JS-библиотеку `@gitbeaker/node` с TypeScript-типами.

## Decision

Используем **REST API через `@gitbeaker/node`** как основной транспорт.

## Rationale

- `@gitbeaker/node` покрывает все нужные эндпоинты с типами — не нужно писать запросы вручную или генерировать типы из схемы
- Для персонального инструмента N+1 запросов не критичны: данные запрашиваются по требованию пользователя, а не в фоне
- REST-документация GitLab значительно полнее GraphQL — меньше сюрпризов при интеграции Draft Notes API

## Trade-offs

GraphQL был бы эффективнее для Diff View (один запрос вместо трёх). Если производительность станет проблемой — конкретные тяжёлые запросы можно перевести на GraphQL без изменения общей архитектуры.
